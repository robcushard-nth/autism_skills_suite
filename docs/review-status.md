# Review Status

This is a living document tracking who has reviewed each skill and what feedback has been incorporated. The suite's credibility depends on transparent review — this file is how that's documented.

**Current overall status: PRE-REVIEW (v0.1.0)**

The skills have been written by the maintainer. They have not yet been reviewed by autistic adults or neurodiversity-affirming clinicians. Use accordingly — these are starter templates for refinement, not finished products.

---

## Review categories

We track review across five categories:

| Category | What it covers |
|---|---|
| **Autistic adult review** | Is the framing right? Does the language land? Do the scripts sound like real autistic experience? |
| **Clinical review** | Is the medical/psychological information accurate? Are the safety guardrails appropriate? |
| **Educational review** | Are the IEP/504 toolkit and school-advocacy materials accurate and current? |
| **Bibliographic review** | Are the citations correct? Are major works missing? Are the audience tags appropriate? |
| **Translation review** | If translated, is the translation faithful and culturally appropriate? |

A skill achieves "reviewed" status (v0.x.0 → v0.x+1.0) when it has at minimum:
- 3+ autistic adult reviewers
- 1+ clinical reviewer
- For parent-coach specifically: 1+ educational reviewer for the IEP material
- For bibliography: 1+ academic reviewer

## How to volunteer as a reviewer

If you'd like to contribute review:

1. Read the skill content in `skills/<skill-name>/`
2. Open a Content Feedback issue with your detailed feedback
3. Or, if you want to do a comprehensive review, email the maintainer to coordinate: rob.cushard@nthsimplesolution.com

We don't require credentials. We do require:
- Honest engagement with the content
- Specific feedback (not just "I don't like it")
- Willingness to be credited (anonymously is fine)

For clinical reviewers, please disclose your professional context (autism researcher, OT, SLP, child psychiatrist, etc.) so we can calibrate the feedback appropriately.

---

## What "reviewed" status changes

When a skill achieves reviewed status:

- Its version increments (v0.1.0 → v0.2.0)
- This document updates with reviewer credits
- The README updates to reflect the new status
- A release is tagged on GitHub

This signals to potential users that the content has had at least one round of community review — but it does not mean the content is final or beyond critique. Ongoing review and updates are expected.
